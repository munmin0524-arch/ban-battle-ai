# PRD: 반배틀 AI — 우리 반이 이긴다

> 최종 수정일: 2026-03-24
> 상태: 프로토타입 기획 단계

---

## 1. 개요

### 1.1 한 줄 요약
**반배틀 AI** — 학급 간 성장 목표를 경쟁하는 리그 시스템. 학생이 주도하고, AI가 트래킹하고, 교사가 감독한다.

### 1.2 배경 및 기획 의도
- 기존 학급 관리는 **교사 주도**로 이루어져 학생 참여도가 낮음
- 학생들이 **직접 목표를 기록하고 경쟁**하는 구조를 통해 자발적 동기 부여
- 학급 단위 팀 경쟁으로 **협동심 + 건전한 경쟁심** 동시 육성
- 독서, 출석 같은 생활 습관뿐 아니라 **학습 대결(퀴즈, 미션)**도 포함하여 재미 요소 강화

### 1.3 대상 사용자
| 구분 | 설명 |
|------|------|
| **주 사용자** | 초등학교 고학년 학생 (4~6학년) |
| **관리자** | 담임 교사 (학급 생성, 목표 설정, 승인) |
| **관전자** | 같은 학교 내 다른 학급 (순위 열람) |

### 1.4 핵심 가치
1. **학생 주도**: 학생이 직접 활동을 기록하고, 반 성적에 기여
2. **팀 경쟁**: 개인이 아닌 학급 단위 경쟁 → 협동 유도
3. **다양한 대결**: 생활 습관(상시) + 퀴즈 배틀(이벤트) + 미션 챌린지(주간)
4. **간편한 운영**: 교사는 엑셀 업로드로 학생 등록, 최소한의 관리

## 3. 학생 등록 흐름

### 3.1 교사 → 학급 생성
```
교사 로그인
  → "새 학급 만들기" 클릭
  → 학급명 입력 (예: "4학년 2반")
  → 엑셀 파일 업로드
  → 미리보기 테이블에서 확인
  → "등록 완료"
  → 학급코드 자동 생성 (예: AB3K7Z)
```

### 3.2 엑셀 양식
| 번호 | 이름 | 성별(선택) |
|------|------|-----------|
| 1 | 김민준 | 남 |
| 2 | 이서연 | 여 |
| ... | ... | ... |

- 첫 번째 행은 헤더 (자동 감지)
- 번호 + 이름만 필수, 나머지는 선택
- 파일 형식: .xlsx, .xls

### 3.3 학생 → 접속
```
학생이 웹 접속
  → 학급코드 입력 (교사가 알려줌)
  → 본인 번호 선택 (드롭다운)
  → 이름 확인 → 입장
```
- 비밀번호 없음 (초등학생 편의, 프로토타입)
- 세션 기반 유지 (브라우저 닫으면 재로그인)

## 4. 대결 시스템

### 4.1 성장 트래킹 리그 (상시)

학급별로 아래 항목의 점수를 **매일** 쌓아 순위를 경쟁한다.

#### 프리셋 목표
| 카테고리 | 단위 | 포인트 | 기록 방식 | 승인 |
|----------|------|--------|-----------|------|
| 독서량 | 권 | 10pt/권 | 학생이 책 제목+권수 입력 | 교사 승인 필요 |
| 출석률 | 일 | 5pt/일 | 학생이 "출석 체크" 버튼 | 자동 승인 |
| 친절 미션 | 회 | 8pt/회 | 학생이 내용 입력 | 교사 승인 필요 |

#### 커스텀 목표
- 교사가 자유롭게 추가 가능
- 예: "일기 쓰기 (5pt/편)", "운동 30분 (7pt/회)"
- 승인 필요 여부 설정 가능

#### 순위 산출
- 학급 점수 = 해당 학급 전체 학생의 승인된 포인트 합산
- 순위는 실시간 갱신
- 학교 내 전체 학급 대상

### 4.2 퀴즈 배틀 (이벤트성)

교사가 만든 퀴즈로 반 vs 반 실시간 대결.

#### 퀴즈 유형
| 유형 | 설명 |
|------|------|
| OX 퀴즈 | 참/거짓 선택 |
| 4지선다 | 4개 보기 중 정답 선택 |

#### 진행 흐름
```
교사: 퀴즈 생성 (문제 5~20개)
  → 참여 학급 선택 (2개 이상)
  → "배틀 시작"

학생: 대기 화면 → 문제 표시 → 답 선택 (제한시간 10~30초)
  → 다음 문제 → ... → 결과 화면
```

#### 점수 산출
- 반별 평균 정답률로 승패 결정
- 승리 반: +50pt (리그 점수에 합산)
- 참여 학생: +10pt (개인 기여)
- 무승부 시: 양 반 +25pt

#### 결과 화면
- 반별 정답률 비교 차트
- 문제별 정답률 상세
- MVP (가장 많이 맞힌 학생)

### 4.3 미션 챌린지 (주간)

매주 새로운 학습 미션을 반별로 도전.

#### 미션 예시
| 미션 | 목표 | 기간 |
|------|------|------|
| 수학 문제 풀기 | 반 합산 200문제 | 1주 |
| 영어 단어 외우기 | 반 합산 500단어 | 1주 |
| 독서 릴레이 | 반 전원 1권씩 | 2주 |
| 과학 실험 보고서 | 반 합산 15편 | 1주 |

#### 진행 방식
- 교사가 미션 생성 (제목, 목표량, 기간)
- 학생이 달성 내역을 기록
- 반별 달성률(%) 실시간 표시
- 미션 달성 시: +30pt (리그 점수에 합산)
- 학교 내 모든 학급이 같은 미션에 도전

## 5. 점수 체계 종합

| 활동 | 포인트 | 빈도 |
|------|--------|------|
| 독서 1권 | 10pt | 상시 |
| 출석 1일 | 5pt | 매일 |
| 친절미션 1회 | 8pt | 상시 |
| 커스텀 목표 | 교사 설정 | 상시 |
| 퀴즈배틀 승리 | 50pt | 이벤트 |
| 퀴즈배틀 참여 | 10pt | 이벤트 |
| 미션챌린지 달성 | 30pt | 주간 |

- 모든 포인트는 **학급 점수에 합산**되어 리그 순위에 반영
- 개인 포인트도 별도 집계 (개인 기여도 확인용)

## 6. 주요 화면 구성

### 6.1 로그인 화면
- 상단: 앱 로고 + "우리반 vs 너네반"
- 교사 로그인: 이름 + 비밀번호
- 학생 로그인: 학급코드 + 번호 선택

### 6.2 대시보드 (메인)
- **반 현황 카드**: 반 이름, 현재 순위, 총 점수, 지난주 대비 변동
- **주간 활동 차트**: 일별 포인트 막대그래프 (카테고리별 색상)
- **목표 진행률**: 카테고리별 프로그레스 바
- **최근 활동 피드**: "[김민준] 독서 2권 기록" 형태의 타임라인

### 6.3 목표 트래킹 페이지
- 카테고리별 탭 (독서 | 출석 | 친절 | 커스텀)
- 활동 기록 폼 (숫자 입력 + 메모)
- 내 기록 히스토리 (승인됨/대기중/반려됨 상태 표시)

### 6.4 리그 순위 페이지
- 전체 학급 순위 테이블 (순위, 반 이름, 점수, 카테고리별 점수)
- 1~3위 메달 표시
- 반 vs 반 비교 카드 (클릭 시)
- 주간 순위 변동 라인 차트

### 6.5 퀴즈 배틀 페이지
- 교사: 퀴즈 생성 폼 (문제, 보기, 정답, 제한시간)
- 학생: 문제 표시 + 타이머 + 보기 버튼
- 결과: 반별 정답률 비교 + MVP

### 6.6 미션 챌린지 페이지
- 현재 진행 중인 미션 카드
- 반별 달성률 프로그레스 바
- 내 기여도 표시

### 6.7 교사 관리 페이지
- 학급 설정 (이름, 학급코드, 학생 명단)
- 엑셀 업로드
- 목표 카테고리 관리
- 활동 승인 큐

## 7. UI/UX 가이드

### 7.1 디자인 톤
- **대상**: 초등 고학년 (4~6학년)
- **톤**: 깔끔하면서 재미있게 (너무 유치하지 않게)
- **컬러**: 파랑(#3B82F6), 주황(#F59E0B), 초록(#10B981), 빨강(#EF4444)
- **모서리**: 둥글게 (border-radius 12px+)
- **레이아웃**: 카드 기반, 여백 넉넉하게
- **반응형**: 모바일 우선 (학생들이 폰으로 접속)

### 7.2 아이콘/시각 요소
- Lucide 아이콘 활용
- 순위: 1위 금메달, 2위 은메달, 3위 동메달
- 진행률: 프로그레스 바 + 퍼센트 숫자
- 차트: 막대그래프(일별), 라인차트(주별 추이)

## 8. 기술 스택

| 구분 | 기술 | 비고 |
|------|------|------|
| 프레임워크 | React + TypeScript | Vite 빌드 |
| 스타일링 | Tailwind CSS | 유틸리티 기반 |
| 차트 | Recharts | 막대/라인/레이더 |
| 아이콘 | Lucide React | |
| 엑셀 파싱 | xlsx (SheetJS) | 학생 명단 업로드 |
| 데이터 | 로컬 상태 + 더미 데이터 | 프로토타입 (백엔드 없음) |

### 8.1 프로토타입 범위
- 백엔드/DB 없음 → React 상태 + 하드코딩된 더미 데이터
- 인증 없음 → 간단한 역할 선택으로 대체
- 실시간 동기화 없음 → 단일 브라우저 세션 내 동작
- **목적**: UI/UX 검증, 기획 의도 확인, 투자/채택 발표용

### 8.2 향후 확장 (프로토타입 이후)
- Firebase 또는 Supabase로 백엔드 전환
- 실시간 퀴즈 배틀 (WebSocket)
- AI 분석 (학급 강점/약점, 개선 추천)
- 뱃지/레벨 시스템 (게이미피케이션)
- 학부모 열람 모드
- 학교 간 리그 확장

| 리스크 | 대응 |
|--------|------|
| 허위 기록 (독서 안 했는데 기록) | 교사 승인 시스템으로 필터링 |

| 개인정보 | 이름+번호만 사용, 외부 공개 없음 |
